Detailed Description

Named, typed multi-parameter container.

ParamSet stores a set of named parameters (Param<T>) using type erasure (ParamBase).

Typical usage: 1) Define a schema (names, types, constraints). 2) Set/get values by name using typed APIs. 3) Serialize/deserialize using JSON or a compact key=value format.

Design notes:

  • Strict parsing: unknown parameters in incoming JSON/compact strings return an error.
  • Deterministic compact serialization: to_compact_string() emits keys sorted lexicographically.
  • Compact-string escaping: the characters '\', ';' and '=' are escaped with a leading '\' inside values.

nlohmann::json jsonmp::ParamSet::to_json ( ) const

Serializes all parameters into a JSON object.

JSON values are emitted with the parameter's declared type:

  • Integer/Unsigned/FloatingPoint -> JSON number
  • Boolean -> JSON boolean
  • String -> JSON string
Note
JSON object key ordering is not part of the contract. Do not rely on ordering.
